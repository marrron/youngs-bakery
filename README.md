#### 커밋 타입

| 커밋     | 타입 설명                                                                                        |
| :------- | :----------------------------------------------------------------------------------------------- |
| Docs     | 문서 수정(md 파일 등)                                                                            |
| Feat     | 새로운 기능 추가                                                                                 |
| Style    | 스타일 변경 (포매팅 수정, 들어쓰기 추가, 빈칸 제거, 스펠링 오류 등)                              |
| Refactor | 리팩토링 작업(코드 동작은 유지하되, 코드의 가독성 및 유지보수성을 향상시키기 위한 내부구조 변경) |
| Fix      | 버그 수정                                                                                        |
| Revert   | 커밋 취소(reset 사용금지)                                                                        |
| Test     | 테스트 코드 추가, 기존 테스트 수정                                                               |
| Build    | 빌드 관련 파일 수정 (패키지 매니저 설정등 개발코드와 무관한 부분)                                |

<br>

#### 커밋 설명 규칙

첫 글자는 항상 대문자로 기재한다
